# DocMind 今日更新日志

## 日期

2026-03-13

## 今日完成内容

### 1. 后端从内存数据源切换为 SQLite

- 引入 `SQLAlchemy` 并接入 `SQLite`
- 新增数据库配置、连接管理和启动初始化逻辑
- 建立 `papers`、`paper_analyses`、`chat_sessions`、`chat_messages`、`tasks` 五类模型
- 让现有 API 从真实数据库读取，而不再依赖内存 mock 数据

### 2. 重构后端分层结构

- 新增 `models`、`repositories`、`services` 目录
- 将论文、分析、聊天、任务相关的数据访问逻辑下沉到 repository
- 将业务编排逻辑收敛到 service 层
- 保持 API 层尽量薄，只负责接收请求和返回响应

### 3. 打通数据库驱动的核心接口

- 保持并验证以下接口可用：
  - `GET /api/papers`
  - `GET /api/papers/{paper_id}`
  - `GET /api/papers/{paper_id}/analysis`
  - `GET /api/papers/{paper_id}/chat`
  - `POST /api/papers/{paper_id}/chat`
  - `POST /api/papers/{paper_id}/summary-tasks`
  - `POST /api/papers/{paper_id}/translation-tasks`
  - `GET /api/tasks/{task_id}`

### 4. 实现导入论文接口

- 新增 `POST /api/papers/import`
- 支持上传 PDF 文件
- 将上传文件保存到本地 `storage/papers`
- 导入后自动创建：
  - 论文记录
  - 分析占位记录
  - 默认聊天会话
  - 初始提示消息
- 当前导入结果状态为“待解析”，符合阶段目标

### 5. 前端切到真实后端接口

- 论文列表、论文详情、分析内容、聊天记录全部改为从后端接口加载
- 问答提交改为调用真实聊天接口
- 前端通过 Vite 代理转发 `/api` 请求到本地 FastAPI
- 不再依赖工作台里的本地 mock 数据驱动主要交互

### 6. 前端接入导入论文按钮

- 工作台右上角“导入论文”按钮已经接到后端上传接口
- 新增轻量上传面板，支持填写：
  - PDF 文件
  - 标题
  - 作者
  - 年份
  - 来源
  - 研究焦点
  - 标签
- 导入成功后自动刷新左侧列表并切换到新论文

### 7. 测试与验证

- 后端测试已扩展到 8 项并通过
- 前端构建已验证通过
- 本地数据库文件和上传目录已加入 `.gitignore`
